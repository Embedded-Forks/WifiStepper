{% mdicon led-off %}Issues & Troubleshoot

Issue Possible Cause Possible Solution
#1 Motor stalls or doesn't move when commanded. The voltage/current limit may be too low. Increase the KT Values (KT Hold, KT Run, KT Accel, KT Decel) to appropriate levels. The values range from 0 to 1 and are read as a perentage (1 = 100%) of the input voltage or current rating depending on the mode selected.
#2 Motor get's too warm/hot after short amount of time. The voltage/current limit may be to high. Decrease the KT Values (KT Hold, KT Run, KT Accel, KT Decel) to appropriate levels. KT values too high may cause damage to the motor. See issue #1 for details about KT Values.
#3 Command Error flag is thrown repeatedly. Some commands can only be given during certian times/modes of operation. Some commands (such as configuration write) can only be given when the motor is not in motion. Other commands (such as ... TODO) can only be given when the motor is not "Busy". For a complete list of command conflicts, see documentation. TODO.
#4 Under Voltage flag is thrown whenever motor is commanded. Your power supply may be entering CC mode and dropping the voltage. The power supply may be hitting the current limits. If the power supply is already at peak current, it may not be powerful enough for your current application. However, first try lowering the KT Values to command less voltage/current.
#5 Over Current flag is thrown whenever motor is commanded The voltage mode jumper may be soldered. The jumper on the bottom side if the board may be soldered over, bypassing the current sense resistor. While it is more efficient running in voltage mode, this jumper must be "open" for current mode. Remove any residual solder bridging the jumper.
#6 Over Current flag is thrown whenever motor is commanded You may be commanding too much current Lower the KT Values to appropriate levels. See issue #2.
